Proxy（代理）是ES6中新增的一个特性。 Proxy 让我们能够以简洁的方式控制外部对 对象的访问。 其功能非常类似于设计模式中的代理模式 使用Proxy的好处是： 对象只需关注于核心逻辑， 一些非核心逻辑（如读取或设置对象某些属性前；记录日志） 可以让proxy来做 。 从而达到关注点分离 。 降级对象复杂度的目的
因为对象内的属性和方法是可以被随意修改的 那么对象的数据就不是确定的， 有风险
1： 通过条件语句来处理 2： 通过属性描述符来禁止对象被修改 3： 通过object 下的defineProperty 来拦截修改 4： 通过Proxy 来代理对象， 也相当于拦截修改
var p = new Proxy(target , handler); target 为 被代理对象。 handler 是一个对象 ， 其handler 是一个对象 。 其声明了target的一些操作， p是代理后的对象 当外界每次对P进行操作时 ， 就会执行handler 对象上的一些方法 。handler 能耐代理的一些常用的方法： get： 获取 set ： 修改 has ： 判断对象是否有该属性 construct ： 构造函数
